## 概要

このガイドは、**DocSplitアプリをクライアント環境に導入する作業者**（システム管理者、SE、社内IT担当者など）を対象としています。

クライアント企業のPCで、Google Cloud Platform（GCP）プロジェクトを作成し、開発者がアプリケーションをデプロイできる環境を準備するための手順を説明します。

### このガイドの対象者

- **導入作業者**: クライアント企業のPCでセットアップ作業を実行する担当者
- **前提知識**: コマンドライン操作の基本（コピー&ペーストができれば十分）
- **作業場所**: クライアント企業のPC（Googleアカウントでログイン可能な環境）

## 事前準備チェックリスト

セットアップ作業を開始する前に、以下を確認してください。

### 1. 開発者から受け取る情報

- [ ] **開発者のメールアドレス**（例: `developer@example.com`）
  - GCPプロジェクトに「オーナー」権限を付与するために必要

### 2. クライアントに用意してもらうもの

- [ ] **Googleアカウント**
  - GCPプロジェクトを作成するためのGoogleアカウント
  - 個人アカウント（`@gmail.com`）または組織アカウント（`@company.com`）

- [ ] **クレジットカード情報**（課金アカウント未作成の場合）
  - GCPの課金アカウント登録に必要
  - 既に課金アカウントがある場合は不要

<div class="warning-box">

**注意**: 組織アカウント（Google Workspace）を使用する場合、組織の管理者がGCPプロジェクト作成を制限している可能性があります。事前に組織の管理者に確認してください。

</div>

## セットアップ方法の選択

以下のフローチャートに従って、適切なセットアップ方法を選択してください。

```
┌───────────────────────────────────┐
│ クライアントのPC環境はどれ？      │
└─────────┬─────────────────────────┘
          │
    ┌─────▼─────┐
    │ Mac/Linux │ → 【推奨】自動スクリプト → setup-automated.md
    └─────┬─────┘
          │
    ┌─────▼─────┐
    │ Windows   │ → 【推奨】自動スクリプト (.bat) → setup-automated.md
    └─────┬─────┘
          │
    ┌─────▼─────────────────────────┐
    │ スクリプトが実行できない？     │
    │ (セキュリティポリシー等)       │
    └─────┬─────────────────────────┘
          │
    ┌─────▼─────┐
    │   Yes     │ → 手動セットアップ → setup-manual.md
    └─────┬─────┘
          │
    ┌─────▼─────┐
    │    No     │ → 自動スクリプト → setup-automated.md
    └───────────┘
```

### 推奨: 自動スクリプト

以下の場合、自動スクリプトを使用してください（最も簡単で確実）:

- Mac、Linux、またはWindowsで、スクリプトファイルを実行できる環境
- コマンドライン操作に不安がある場合（スクリプトが自動で実行します）

**👉 [自動スクリプトによるセットアップ](setup-automated.md)**

---

### 手動セットアップ（制限環境向け）

以下の場合、手動セットアップを選択してください:

- 企業のセキュリティポリシーで、スクリプト実行が禁止されている
- スクリプトがエラーで動作しない
- 各ステップを手動で確認しながら進めたい

**👉 [手動セットアップ](setup-manual.md)**
